What is Ordinary Least Squares?

Ordinary least squares (OLS), or simply least squares, seeks to find a vector x^∈RN\hat{\mathbf{x}} \in \mathbb{R}^Nx^∈RN that, after being multiplied by a matrix A∈RM×N\mathbf{A} \in \mathbb{R}^{M \times N}A∈RM×N, minimizes the distance to another vector y∈RM\mathbf{y} \in \mathbb{R}^My∈RM. In math, OLS seeks to solve the following optimization problem:

x^=argminx f(x),f(x)=12∥Ax−y∥22, \hat{\mathbf{x}} = \text{argmin}_{\mathbf{x}}\; f(\mathbf{x}), \qquad f(\mathbf{x}) = \frac{1}{2} \|\mathbf{A}\mathbf{x} - \mathbf{y}\|_2^2, x^=argminx​f(x),f(x)=21​∥Ax−y∥22​,

where ∥x∥22=x′x=∑i=1Nxi2\|\mathbf{x}\|_2^2 = \mathbf{x}'\mathbf{x} = \sum_{i = 1}^N x_i^2∥x∥22​=x′x=∑i=1N​xi2​, and x′\mathbf{x}'x′ is the transpose of x\mathbf{x}x.

Taking a systems perspective, A\mathbf{A}A could characterize a linear system, y\mathbf{y}y could be observed data from that system, and x^\hat{\mathbf{x}}x^ would then be the most likely input that caused y\mathbf{y}y.

OLS Solution

The solution to (1) is straightforward:

x^=A+y, \hat{\mathbf{x}} = \mathbf{A}^+ \mathbf{y}, x^=A+y,

where A+\mathbf{A}^+A+ is the pseudoinverse of A\mathbf{A}A.

Solving OLS in Julia

Solving OLS problems in Julia is likewise easy, and can be done with the following function.

function ols(A::AbstractMatrix{<:Real}, y::AbstractVector{<:Real})
    x̂ = A \ y
    return x̂
end

Example Application: Fitting a Quadratic Function to Noisy Data

Suppose we have data y\mathbf{y}y that contains noisy observations from an unknown function that we suspect is quadratic. Further suppose that we know at what time each observation in y\mathbf{y}y was acquired (so yiy_iyi​ was acquired at time tit_iti​). We can use OLS to fit a quadratic function of time to the data. Each data point yiy_iyi​ can be modeled as being approximately equal to a⋅ti2+b⋅ti+ca \cdot t_i^2 + b \cdot t_i + ca⋅ti2​+b⋅ti​+c. We can use this model to specify what A\mathbf{A}A and x\mathbf{x}x are. In this case, aaa, bbb, and ccc are unknown, but we want to determine what they are; therefore, x=[a,b,c]′\mathbf{x} = [a, b, c]'x=[a,b,c]′. Then A\mathbf{A}A must contain the tit_iti​'s:

[t12t11⋮tM2tM1]. \begin{bmatrix} t_1^2 & t_1 & 1 \\ & \vdots & \\ t_M^2 & t_M & 1 \\ \end{bmatrix}. ⎣⎢⎢⎡​t12​tM2​​t1​⋮tM​​11​⎦⎥⎥⎤​.

(If you're not convinced this is correct, take one of the rows of A\mathbf{A}A and multiply it by x\mathbf{x}x and see how it compares to our model for yiy_iyi​.)

Code

Now let's put it all together in code.

function fit_quadratic(t::AbstractVector{<:Real}, y::AbstractVector{<:Real})
    A = [t.^2 t ones(eltype(t), length(t))]
    (â, b̂, ĉ) = ols(A, y)
    return (â, b̂, ĉ)
end

using Random
Random.seed!(0)
(a, b, c) = (1, -1, 1//4) # True parameters
f(t) = a * t^2 + b * t + c
M = 50 # Number of observations
t = rand(M)
y = f.(t) .+ 0.01 .* randn(M) # Noisy observations
(â, b̂, ĉ) = fit_quadratic(t, y)

Let's see how the estimated parameters compare to the true parameters.

(a, b, c) = (1.000, -1.000, 0.250)
(â, b̂, ĉ) = (0.989, -0.995, 0.249)

The estimated parameters agree pretty well with the true values! Let's visualize how our quadratic fit compares to the noisy data and to the underlying true quadratic curve.

using Plots
f̂(t) = â * t^2 + b̂ * t + ĉ
scatter(t, y, label = "Observed Data")
plot!(f, LinRange(0, 1, 101), label = "True Curve", line = (2, :red))
plot!(f̂, LinRange(0, 1, 101), label = "Estimated Curve", line = (2, :blue, :dash))
plot!(xlabel = "t", ylabel = "y")

As expected from how close our estimated parameters were to the true parameters, our predicted quadratic curve almost exactly coincides with the true quadratic curve that produced the observed measurements.

FAQ

Where does A\mathbf{A}A come from? How do we know what it is?

Generally, A\mathbf{A}A either comes from data, in which case finding A\mathbf{A}A is simply a matter of reading data from a file, or A\mathbf{A}A models a system that transforms inputs x\mathbf{x}x into outputs y\mathbf{y}y. In this latter case, determining A\mathbf{A}A boils down to learning about the system you want to model and then converting the physics and/or assumptions into matrix form. In the quadratic fitting example, we had to create A\mathbf{A}A ourselves, and we did so by using our knowledge of quadratic functions and how they map inputs to outputs. As another example, in magnetic resonance imaging (MRI) the system matrix A\mathbf{A}A corresponds to the discrete fourier transform (DFT) matrix; again, determining A\mathbf{A}A came from understanding MRI physics and then converting it into matrix form.
