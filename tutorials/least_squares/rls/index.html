What is Regularized Least Squares?

Regularized least squares (RLS) is ordinary least squares (OLS) but with an extra regularization term. Described as an optimization problem, RLS solves

x^=argminx f(x)+βR(x). \hat{\mathbf{x}} = \text{argmin}_{\mathbf{x}}\; f(\mathbf{x}) + \beta R(\mathbf{x}). x^=argminx​f(x)+βR(x).

As in OLS, f(x)=12∥Ax−y∥22f(\mathbf{x}) = \frac{1}{2} \|\mathbf{A} \mathbf{x} - \mathbf{y}\|_2^2f(x)=21​∥Ax−y∥22​, with x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN, A∈RM×N\mathbf{A} \in \mathbb{R}^{M \times N}A∈RM×N, and y∈RM\mathbf{y} \in \mathbb{R}^My∈RM. Unlike OLS, in RLS we are also trying to minimize the regularizer R(x)R(\mathbf{x})R(x). The regularization parameter β>0\beta > 0β>0 is a user-controlled parameter that controls the relative importance of minimizing R(x)R(\mathbf{x})R(x) versus f(x)f(\mathbf{x})f(x); the smaller β\betaβ is, the more emphasis is placed on minimizing f(x)f(\mathbf{x})f(x), and the larger β\betaβ is, the more emphasis is placed on minimizing R(x)R(\mathbf{x})R(x).

Tikhonov Regularization

The most common choice of regularizer is called Tikhonov regularization, where

R(x)=12∥x∥22. R(\mathbf{x}) = \frac{1}{2} \|\mathbf{x}\|_2^2. R(x)=21​∥x∥22​.

In RLS with Tikhonov regularization, we want to find x^\hat{\mathbf{x}}x^ such that Ax^\mathbf{A}\hat{\mathbf{x}}Ax^ is close to y\mathbf{y}y (minimizing f(x)f(\mathbf{x})f(x)), but at the same time we don't want the elements of x^\hat{\mathbf{x}}x^ to be too large (minimizing R(x)R(\mathbf{x})R(x)).

RLS Solution: Tikhonov Regularization

RLS with Tikhonov regularization is actually OLS in disguise, so we can use the solution to OLS. To do so, we need to rewrite the function we want to minimize so that it looks like OLS. We can do so as follows (in the following, I\mathbf{I}I is the identity matrix, and 0\mathbf{0}0 is a vector of all zeros):

f(x)+βR(x)=12∥Ax−y∥22+β2∥x∥22=12∑i=1M([Ax]i−yi)2+β2∑j=1Nxj2=12∑i=1M([Ax]i−yi)2+12∑j=1N([βIx]j−0)2=12∑i=1M+N([AxβIx]i−[y0]i)2=12∑i=1M+N([A~x]i−y~i)2=12∥A~x−y~∥22\begin{aligned} f(\mathbf{x}) + \beta R(\mathbf{x}) &= \frac{1}{2} \|\mathbf{A}\mathbf{x} - \mathbf{y}\|_2^2 + \frac{\beta}{2} \|\mathbf{x}\|_2^2 \\ &= \frac{1}{2} \sum_{i = 1}^M ([\mathbf{A}\mathbf{x}]_i - y_i)^2 + \frac{\beta}{2} \sum_{j = 1}^N x_j^2 \\ &= \frac{1}{2} \sum_{i = 1}^M ([\mathbf{A}\mathbf{x}]_i - y_i)^2 + \frac{1}{2} \sum_{j = 1}^N ([\sqrt{\beta}\mathbf{I}\mathbf{x}]_j - 0)^2 \\ &= \frac{1}{2} \sum_{i = 1}^{M + N} \left(\begin{bmatrix} \mathbf{A}\mathbf{x} \\ \sqrt{\beta}\mathbf{I}\mathbf{x} \end{bmatrix}_i - \begin{bmatrix} \mathbf{y} \\ \mathbf{0} \end{bmatrix}_i\right)^2 \\ &= \frac{1}{2} \sum_{i = 1}^{M + N} ([\tilde{\mathbf{A}}\mathbf{x}]_i - \tilde{y}_i)^2 \\ &= \frac{1}{2} \|\tilde{\mathbf{A}}\mathbf{x} - \tilde{\mathbf{y}}\|_2^2 \\ \end{aligned}f(x)+βR(x)​=21​∥Ax−y∥22​+2β​∥x∥22​=21​i=1∑M​([Ax]i​−yi​)2+2β​j=1∑N​xj2​=21​i=1∑M​([Ax]i​−yi​)2+21​j=1∑N​([β​Ix]j​−0)2=21​i=1∑M+N​([Axβ​Ix​]i​−[y0​]i​)2=21​i=1∑M+N​([A~x]i​−y~​i​)2=21​∥A~x−y~​∥22​​

Now we see that RLS with Tikhonov regularization is the same as OLS with

A~=[AβI] \tilde{\mathbf{A}} = \begin{bmatrix} \mathbf{A} \\ \sqrt{\beta} \mathbf{I} \end{bmatrix} A~=[Aβ​I​]

instead of just A\mathbf{A}A, and

y~=[y0] \tilde{\mathbf{y}} = \begin{bmatrix} \mathbf{y} \\ \mathbf{0} \end{bmatrix} y~​=[y0​]

instead of just y\mathbf{y}y.

Solving RLS with Tikhonov Regularization in Julia

Solving RLS with Tikhonov regularization in Julia is as simple as calling ols with A~\tilde{\mathbf{A}}A~ and y~\tilde{\mathbf{y}}y~​.

using LinearAlgebra: I

function ols(A::AbstractMatrix{<:Real}, y::AbstractVector{<:Real})
    x̂ = A \ y
    return x̂
end

function rls(A::AbstractMatrix{<:Real}, y::AbstractVector{<:Real}; β::Real = 1)
    Ã = [A; sqrt(β) * I]
    ỹ = [y; zeros(eltype(y), size(A, 2))]
    x̂ = ols(Ã, ỹ)
    return x̂
end

Example Application: Image Denoising

Suppose we have a noisy image y\mathbf{y}y and we want to clean it up. One way to do this is by solving the following optimization problem:

x^=argminx 12∥x−y∥22+β2∥x∥22, \hat{\mathbf{x}} = \text{argmin}_{x}\; \frac{1}{2} \|\mathbf{x} - \mathbf{y}\|_2^2 + \frac{\beta}{2} \|\mathbf{x}\|_2^2, x^=argminx​21​∥x−y∥22​+2β​∥x∥22​,

i.e., RLS with A=I\mathbf{A} = \mathbf{I}A=I. In this case, x^\hat{\mathbf{x}}x^ is the cleaned up image. The 12∥x−y∥22\frac{1}{2} \|\mathbf{x} - \mathbf{y}\|_2^221​∥x−y∥22​ term tries to find an image that is close to the noisy image, but the β2∥x∥22\frac{\beta}{2} \|\mathbf{x}\|_2^22β​∥x∥22​ term tries to prevent the cleaned up image from keeping too much noise.

Normally, we think of an image as a matrix of numbers, so we'll first need to vectorize the image. After finding x^\hat{\mathbf{x}}x^, we will reshape it into a matrix for display.

Code

Let's see image denoising in action.

using LinearAlgebra: I
using Statistics: mean

function denoise_image(image::AbstractMatrix{<:Real}; β::Real = 1)
    image_mean = mean(image)
    image_zero_mean = image .- image_mean
    clean_vector = rls(I(length(image)), vec(image_zero_mean); β)
    clean_image = reshape(clean_vector, size(image)) .+ image_mean
    return clean_image
end

# Set up the true image
using Images: channelview, Gray, imresize
using TestImages: testimage
image_highres = Float64.(channelview(Gray.(testimage("peppers_gray"))))
image = imresize(image_highres, (64, 64))

# Add noise, then clean it up
using Random
Random.seed!(0)
noisy_image = image .+ 0.05 .* randn(size(image)...)
clean_image = denoise_image(noisy_image; β = 0.5)

# Display results
using Images: mosaicview, clamp01nan
mosaic = map(clamp01nan, mosaicview(image, noisy_image, clean_image; nrow = 1))

The left image is the clean image, the middle is the noisy image, and the right is the denoised image. We can see the right image has less variation in pixel intensity due to noise than the middle image, but the tradeoff is that the whole image seems to have less dynamic range. Personally, I think the middle image looks better than the one on the right, but humans tend to be good at seeing through noise, so maybe the right image would be easier for a computer to analyze.
